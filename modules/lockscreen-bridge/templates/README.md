# Templates

Source files for the iOS WidgetKit Extension target. These are NOT generated by `expo prebuild` — they're authored here and copied into `ios/MotoILWidget/` at prebuild time by the plugin (step-3 enhancement).

## Files

| file | role |
|---|---|
| `ios/MotoILWidget/Info.plist` | Extension `Info.plist`. `NSExtensionPointIdentifier = com.apple.widgetkit-extension`. Standard `$()` substitution for bundle id/versions filled in by Xcode from build settings (which `withWidgetExtensionTarget` configures). |
| `ios/MotoILWidget/MotoILWidget.entitlements` | Extension `.entitlements`. Mirrors the App Group on the main app target so both can read/write the same `UserDefaults(suiteName: "group.com.echcomp.motoil.ice")`. |
| `ios/MotoILWidget/IceSnapshot.swift` | Codable types mirroring `src/features/lockscreen/types.ts` schema v1. Explicit `CodingKeys` so a Swift rename can't silently break the JSON contract with the RN writer. |
| `ios/MotoILWidget/SnapshotReader.swift` | Reads base64 blob from App Group `UserDefaults`, fetches AES key from shared Keychain access group (`kSecAttrAccessibleAfterFirstUnlock`), AES-256-GCM decrypts via CryptoKit, JSON-decodes, validates `schema_version <= knownSchemaVersion`. Returns a `Result<IceSnapshot, SnapshotReadError>`. |
| `ios/MotoILWidget/Provider.swift` | `TimelineProvider`. Maps the reader result to one of five `IceWidgetState` cases (ready/empty/stale/corrupt/locked) per `widget-ui.md §5`. Hourly fallback timeline; RN-side writer calls `WidgetCenter.reloadAllTimelines()` on every successful write (step 3c). |
| `ios/MotoILWidget/WidgetView.swift` | SwiftUI rendering. iOS-16-safe — no `containerBackground` or `contentMarginsDisabled` (both iOS 17+). Brand red `#E53935` for the blood-type badge; phone numbers force LTR via `.environment(\.layoutDirection, .leftToRight)` so they don't mirror under Hebrew RTL. |
| `ios/MotoILWidget/MotoILWidget.swift` | `Widget` config — `kind`, `configurationDisplayName`, `description`, `supportedFamilies`. |
| `ios/MotoILWidget/MotoILWidgetBundle.swift` | `@main WidgetBundle` entry point. |

## When these get copied

When `iosWidgetExtensionEnabled: true` is passed to the plugin, `src/withWidgetExtensionResources.js` recursively copies `templates/ios/MotoILWidget/` to `<platformProjectRoot>/MotoILWidget/` via `withDangerousMod` BEFORE `withWidgetExtensionTarget` runs the Xcode mods. This ordering matters: the Xcode-project mod registers build phases against files that must already exist on disk, and `warnIfSwiftMissing` inside that mod only stays quiet when the files are there.

When `iosWidgetExtensionEnabled: false` (default), neither mod runs and `ios/MotoILWidget/` does not appear in prebuild output.

`force: true` on the copy makes it idempotent — re-running `expo prebuild --clean` always overwrites the destination from this directory, so the templates here are the source of truth and any drift in `ios/MotoILWidget/` between prebuild runs is wiped.

## Verification

All six Swift files type-check cleanly against the iOS 16 SDK with WidgetKit + SwiftUI + CryptoKit + Security:

```bash
cd modules/lockscreen-bridge/templates/ios/MotoILWidget
swiftc -typecheck \
  -sdk "$(xcrun --sdk iphoneos --show-sdk-path)" \
  -target arm64-apple-ios16.0 \
  -framework WidgetKit -framework SwiftUI -framework CryptoKit -framework Security \
  IceSnapshot.swift SnapshotReader.swift Provider.swift WidgetView.swift MotoILWidget.swift MotoILWidgetBundle.swift
```

Full build verification still requires Xcode on a real device — that lands in the §6 DoD checklist of the integration PR.

## Why not check in `ios/MotoILWidget/` directly

Project rule (`docs/HANDOFF.md` and Slack confirmation from PM): `ios/` and `android/` are prebuild artifacts. Anything checked into `ios/MotoILWidget/` would be blown away by `expo prebuild --clean`. The plugin must be the canonical source.
