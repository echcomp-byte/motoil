# Templates

Source files for the iOS WidgetKit Extension target. These are NOT generated by `expo prebuild` — they're authored here and copied into `ios/MotoILWidget/` at prebuild time by the plugin (step-3 enhancement).

## Files

- `ios/MotoILWidget/Info.plist` — extension `Info.plist`. Declares `NSExtensionPointIdentifier = com.apple.widgetkit-extension`. Standard `$()` substitution placeholders for bundle id / versions are filled in by Xcode at build time from the target's build settings (which `withWidgetExtensionTarget` configures).
- `ios/MotoILWidget/MotoILWidget.entitlements` — extension `.entitlements`. Mirrors the App Group entitlement on the main app target so both can read/write the same `UserDefaults(suiteName: "group.com.echcomp.motoil.ice")`.

## When these get copied

Today: never. `withWidgetExtensionTarget.js` declares a Sources/Frameworks/Resources build-phase shape, but doesn't yet copy these source files. The plugin's `warnIfSwiftMissing()` will flag their absence at `ios/MotoILWidget/` when `iosWidgetExtensionEnabled: true`.

Step-3 plugin enhancement (post RFC #2 signoff) will add:

```js
// modules/lockscreen-bridge/src/withWidgetExtensionResources.js (TBD)

```

## Swift sources

`MotoILWidget.swift` (the actual SwiftUI views + WidgetBundle) is NOT scaffolded here. It lands as part of step 3 when RFC #2 schema v1 is frozen. Until then there is no stable shape for the snapshot reader to compile against. The template directory above will get a sibling `MotoILWidget.swift` and a `MotoILWidgetBundle.swift` at that point.

## Why not check in `ios/MotoILWidget/` directly

Project rule (`docs/HANDOFF.md` and Slack confirmation from PM): `ios/` and `android/` are prebuild artifacts. Anything checked into `ios/MotoILWidget/` would be blown away by `expo prebuild --clean`. The plugin must be the canonical source.
